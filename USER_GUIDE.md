## 快速开始

### 第一次使用前

1. 打开 SysSubtitle → 点右上角 **⚙️ 设置**
2. 进入 **API** 标签页
3. 填入 **Deepgram API Key**（语音识别必需）
4. 填入翻译引擎 API Key（DeepL / Claude / Google 任选一个）
5. 返回主界面，选择源语言和目标语言
6. 点击 **开始** 按钮

### 获取 API Key

| 服务 | 注册地址 | 免费额度 |
|------|---------|---------|
| Deepgram | https://console.deepgram.com | $200 免费额度 |
| DeepL | https://www.deepl.com/pro-api | 每月 50 万字符 |
| Google Translate | https://console.cloud.google.com | $300 试用金 |
| Claude | https://console.anthropic.com | 按量付费 |

## 两种音频模式

### 📡 屏幕录制模式 (Broadcast)

**适用场景**: 看视频、听播客、视频通话 — 直接捕获系统内部音频，音质最佳。

**启动方式**:
1. 设置中选择 **📡 屏幕录制**
2. 点击设置中的 **开始录屏** 按钮（或：iOS 控制中心 → 长按录屏按钮 → 选择 **SysSubtitle Broadcast**）
3. 回到主界面点 **开始**
4. 切换到你要看的视频/播客 app

**注意**: 
- 屏幕录制会在状态栏显示红色/紫色指示条，这是 iOS 系统行为
- 录制中无法切换音频模式，需先停止

### 🎙 麦克风模式 (Microphone)

**适用场景**: 面对面对话、会议、讲座 — 或者不方便开启屏幕录制时，外放声音让麦克风拾取。

**启动方式**:
1. 设置中选择 **🎙 麦克风**
2. 回到主界面点 **开始**
3. 首次使用会请求麦克风权限，请允许

**注意**:
- 麦克风模式拾取环境声音，安静环境效果最好
- 外放音量越大，识别越准确

## 基本操作流程

### 1. 选择语言

主界面顶部：
```
[源语言] → [目标语言]
 日本語  →   中文
```
点击可切换。支持的语言包括：日语、韩语、英语、中文。

### 2. 选择识别引擎

设置 → 语音识别：

| 引擎 | 标识 | 说明 |
|------|------|------|
| Deepgram Solo | DG·S | 单人模式，适合独白、演讲、新闻 |
| Deepgram Multi | DG·M | 多人模式，自动识别不同说话人，用颜色区分 |
| Apple Speech | AS | 系统内置，免费，无需 API Key |

**推荐**: 日常使用选 **DG·M**（多人模式），即使只有一个人说话也能正常工作。

### 3. 选择翻译引擎

设置 → 翻译引擎：

| 引擎 | 特点 |
|------|------|
| Apple 翻译 | 免费，需下载离线语言包 |
| DeepL | 翻译质量高，推荐 |
| Claude | 支持上下文精翻 |
| Google | 语言覆盖广 |

### 4. 开始/停止

- 点击底部 **▶ 开始** → 开始识别和翻译
- 字幕实时显示在主界面
- 点击 **■ 停止** → 结束，自动保存会话

### 5. 阅读字幕

```
🔵A  そうですね一般的ではないんですよね
     嗯，确实不太寻常呢

🟠B  僕も同じだったんですよ
     我也一样
```

- 字母 A/B/C... 表示不同说话人
- 颜色自动分配（蓝/橙/绿/粉/紫/黄）
- 上方是原文，下方是译文

## 设置详解

### 语速预设 (DG Multi 模式)

| 预设 | 适用场景 | 特点 |
|------|---------|------|
| 🐢 慢 | 演讲、正式场合 | 句子更完整，延迟稍高 |
| 🚶 中 | 播客、日常对话 | 平衡（推荐） |
| 🐇 快 | 快速对话、综艺 | 响应快，句子较短 |

录制中可以切换语速，不需要停止。

### 录音保存

设置 → 文档标签页 → **保存录音**：开启后停止时自动保存 WAV 文件。

录音用途：
- 事后回听
- REST 说话人修正（更准确的说话人识别）
- 分享导出

### Track B 说话人校准

开启后，录制过程中每 30-60 秒自动用 Deepgram REST API 重新分析说话人，修正实时识别的说话人标记。

设置 → 文档标签页 → **Track B** → 开启

## PiP 画中画

### 开启方法

1. 设置 → 显示标签页 → **PiP 画中画** → 开启
2. 开始录制后，PiP 悬浮窗自动出现
3. 切换到其他 App，字幕浮窗保持显示

### 窗口布局

```
┌────────────────────────────┐
│  历史译文 (淡色)            │
│                            │
│  🔵A 最后一句原文 (白色)    │
│      最后一句译文 (说话人色) │
│                            │
│  正在说的话... (灰色)       │
│  快速翻译预览 (淡色)        │
└────────────────────────────┘
```

### 操作

- **拖动**: 移动悬浮窗位置
- **捏合**: 调整大小（小/中/大）
- **双击**: 返回 App

## 会话管理

### 查看历史会话

设置 → 文档标签页 → **会话历史** → 点击任一会话

### 会话详情 4 个标签页

| 标签 | 内容 |
|------|------|
| 📝 转录 | 完整的原文+译文，带说话人颜色 |
| 🔧 修正 | 翻译层级分布、说话人修正详情、REST 修正按钮 |
| 🤖 AI | AI 精翻（用 LLM 重新翻译全文）、AI 摘要 |
| 🐛 诊断 | 详细的技术日志，用于排查问题 |

### REST 说话人修正

在修正标签页中，如果会话有保存录音：
1. 点击 **REST 说话人修正** 按钮
2. 等待上传和分析（通常 10-30 秒）
3. 完成后说话人标记会更准确

### 导出

会话详情右上角 **分享按钮**：
- 导出转录文本
- 导出诊断报告

## API Key 配置

### Deepgram（语音识别 — 必需）

1. 访问 https://console.deepgram.com
2. 注册账号（有 $200 免费额度）
3. 创建 API Key
4. 复制 Key → 粘贴到 SysSubtitle 设置 → API → Deepgram

**多 Key 轮转**: 用逗号分隔多个 Key，用完一个自动切换下一个。

### DeepL（翻译 — 推荐）

1. 访问 https://www.deepl.com/pro-api
2. 注册 Free 账号（每月 50 万字符免费）
3. 在账户设置中找到 API Key
4. 复制 → 粘贴到设置 → API → DeepL

### 测试 Key

设置 → API 标签页 → 每个 Key 旁边有 **测试** 按钮，点击可验证 Key 是否有效。

## 常见问题

### Q: 没有声音/没有字幕出现？

**屏幕录制模式**:
- 确认已从控制中心启动屏幕录制，并选择了 SysSubtitle Broadcast
- 状态栏应显示红色/紫色录制指示条
- 确认视频/播客正在播放且有声音

**麦克风模式**:
- 确认已允许麦克风权限（iOS 设置 → SysSubtitle → 麦克风）
- 确认设备扬声器有声音输出
- 尝试提高音量

### Q: 有原文但没有翻译？

- 检查是否已配置翻译引擎的 API Key
- 设置 → API → 点击测试按钮验证
- 如果使用 Apple 翻译：iOS 设置 → 通用 → 翻译 → 确认已下载对应语言包

### Q: 说话人识别不准确？

- 使用 **DG·M 多人模式**
- 开启 **Track B** 自动校准
- 录制结束后在会话详情中使用 **REST 说话人修正**
- 使用 🐢 慢速预设可提高准确度

### Q: 字幕延迟太高？

- 切换到 🐇 快速预设
- 使用 DeepL 翻译（比 Claude 更快）
- 确保网络连接稳定

### Q: 屏幕录制模式下切换 App 后字幕消失？

- 开启 PiP 画中画功能
- 设置 → 显示 → PiP 画中画 → 开启

### Q: 如何停止屏幕录制？

三种方式：
1. 回到 SysSubtitle App → 点停止按钮（会同时停止 Broadcast）
2. 点击状态栏的红色录制指示条 → 确认停止
3. 控制中心 → 点击录屏按钮

### Q: 杀掉 App 后屏幕录制还在继续？

正常情况下 App 被杀后 30 秒内 Broadcast 会自动停止。如果没有，手动点击状态栏红色指示条停止。
